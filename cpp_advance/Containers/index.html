    Choosing Sequence Containers¶

    â— Sequence containers are for when you need to enforce some order on your information!

    â— Can usually use an std::vector for most anything

    â— If you need particularly fast inserts in the front, consider an std::deque

    â— For joining/working with multiple lists, consider an std::list (very rarely)

    Map implementation¶

    Maps are implemented with pairs! (std::pair<const key, value>)

    â— Note the const! Keys must be immutable(ä¸å¯å˜çš„).

    â— Indexing into the map (myMap[key]) searches through the underlying collection of pairs first attribute for the key and will return its second attribute.

    Unordered maps/sets¶

    Both maps and sets in the STL have an unordered version!

    â— Ordered maps/sets require a comparison operator to be defined.

    â— Unordered maps/sets require a hash function to be defined.
